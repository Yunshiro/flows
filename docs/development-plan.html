1. 项目概述与目标

1.1 项目定位

Flows 是一款面向个人使用的跨平台桌面时间管理工具，帮助用户规划每日任务、记录工作复盘、自动生成周报，并管理个人 Markdown 知识库。

1.2 核心目标

  • 本地优先：所有数据存储在用户本地设备，隐私可控。
  • AI 驱动：利用 LLM 自动总结周报，减少手动记录负担。
  • 极简高效：聚焦核心时间管理流程，避免功能膨胀。
  • 数据可迁移：支持 Git 同步，数据不绑定平台。

1.3 非目标（范围外）

  • 非多人在线协作工具（如 Notion、飞书）
  • 非企业级项目管理（如 Jira、Linear）
  • 非移动端优先（仅桌面端）

3.4 UI 框架选型

推荐方案：TDesign Vue Next — 腾讯出品企业级 Vue 3 组件库，设计规范、TypeScript 友好、主题可定制。

备选方案：Naive UI / Element Plus / Ant Design Vue / Radix Vue / PrimeVue / Vuetify 等均可。

选型理由：TDesign 是国内 Vue 生态最成熟的企业级组件库，社区活跃、文档齐全、更新及时。

6. 模块详细设计

6.1 模块一：每日待办清单

页面布局


┌──────────────────────────────────────┐
│  日期选择器    │  操作栏              │
│  [◀ 前一天]    │  [+ 新增待办]        │
│  [今天  ▶]    │  [拖拽排序] [筛选]   │
└──────────────────────────────────────┘

左侧为日历日期选择器，可快速切换到历史某天查看待办；顶部操作栏包含新增按钮、筛选下拉框（按优先级、标签、完成状态）和搜索输入框；待办列表每一项左侧有拖拽手柄，支持拖拽排序，点击左侧复选框标记完成。

交互流程

  1. 页面加载时读取今日待办列表（SQLite 查询），按未完成优先、创建时间倒序排列。
  2. 用户点击 新增待办 按钮 → 弹出编辑表单（标题、优先级、预计用时、标签），填写后点击保存。
  3. 表单校验：标题必填、预计用时可选填。
  4. 保存成功后刷新列表，表单清空。
  5. 点击待办卡片左侧复选框 → 切换完成状态，自动记录实际用时和完成时间。
  6. 未完成项在切换日期时弹窗询问是否滚动到次日。
  7. 右侧删除按钮 hover 显示，点击后二次确认删除。
  8. 筛选下拉变更时自动刷新列表。

自定义扩展

  • 支持标签分类自定义颜色。
  • 支持每日复盘模板自定义（默认显示今日日期，自动关联已完成待办）。

6.2 模块二：每日复盘小记

页面布局


┌──────────────────────────────────────┐
│  日期显示（只读）  │  Markdown 编辑器   │
│  [2026-06-25]     │  [复盘内容...]      │
│  心情: 高效       │  [保存] [时间线]    │

顶部显示当前日期；编辑器支持 Markdown 实时预览；心情标记按钮组切换；下方操作栏包含保存按钮和跳转时间线图标；时间线视图为垂直卡片流，展示历史每日标题、心情图标和关联待办标签。

交互流程

  1. 页面加载时读取今日复盘（SQLite 按日期查询）。
  2. 编辑器支持 Markdown 语法高亮、加粗、斜体等快捷操作。
  3. 预览面板实时渲染 Markdown。
  4. 点击保存——内容写入 SQLite，同时关联待办 ID 列表更新。
  5. 保存成功后跳转至时间线视图。

6.3 模块三：每周自动总结

页面布局


┌──────────────────────────────────────────┐
│  周选择器        │  AI 总结卡片          │
│  [◀ 前一周]      │  [本周: 06/23 - 06/29] │
│  [本周 ▶]       │  AI 生成中...  [重试]  │
│                  │  [手动编辑] [导出]    │
└──────────────────────────────────────────┘

顶部周选择器可快速切换历史周；下方本周卡片显示日期范围和 AI 生成的总结正文，包含本周完成情况统计；底部操作栏包含重试生成按钮、手动编辑区和导出入口。

交互流程

  1. 页面加载时查询本周每日复盘和待办数据（SQLite），提交给 LLM API 生成总结。
  2. AI 生成内容以卡片形式展示，包含本周统计图表（完成率饼图、用时分布）。
  3. 用户点击重试按钮 → 重新请求 LLM。
  4. 用户手动编辑总结正文。
  5. 点击导出 → 复制 Markdown 到剪贴板 / 保存为本地文件。

6.4 模块四：Markdown 笔记

页面布局


┌────────────────────────────────────────┐
│  文件树面板      │  Markdown 编辑器     │
│  [📁 文件夹]     │  [标题 / 正文...]     │
│  [📄 笔记列表]   │  [实时预览] [大纲]   │
└────────────────────────────────────────┘

左侧为文件树面板，展示文件夹层级结构；右侧为编辑器区域，包含标题输入框、正文编辑区和实时预览面板；底部为大纲视图（显示标题层级）。

交互流程

  1. 页面加载时读取本地文件系统目录树。
  2. 用户点击文件夹 → 展开/折叠子目录；点击笔记文件 → 在右侧编辑器中打开。
  3. 编辑器支持 Markdown 语法高亮、加粗、斜体、代码块等快捷操作。
  4. 预览面板实时渲染 Markdown，支持 GitHub Flavored Markdown（GFM）。
  5. 大纲视图跟随编辑器滚动，点击标题跳转至对应正文位置。
  6. 点击保存——内容写入本地文件系统，更新文件树列表。
  7. 全文搜索（Ctrl+K / Cmd+K）——在文件树面板顶部搜索框输入关键词，回车后高亮匹配的笔记文件，点击跳转至编辑器对应位置。
  8. 双向链接 [[wiki-link]] ——编辑器内输入双方括号自动解析为可点击的内部链接，点击后跳转至目标笔记文件。

7.2 详细设计

  1. 笔记目录默认路径：{$HOME}/flows-notes/。
  2. 初始化时若目录不存在则自动创建。
  3. 用户在设置页配置远程仓库 URL 和分支名。
  4. 首次同步前需手动点击推送，后续可开启定时自动拉取。
  5. 每次同步前进行冲突检测：远端有新提交时先拉取再合并，使用 rebase 策略（保留本地修改优先）。
  6. 同步完成后刷新笔记文件树。
  7. 离线时允许手动编辑笔记，在线恢复后自动推送。
